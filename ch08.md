# 第 8 章 實戰演練：從 0 開始打造 PDF 翻譯器

## 8-1 用簡單提示詞生成程式

### 用 Plan Mode 定義需求

- 輸入的提示內容：

  ```md
  幫我製作一個 PDF 翻譯器的 app，包含以下要求：

  1. 以 Python 製作
  2. 盡可能支援多種語言翻譯成繁體中文
  3. 使用 deep-translator 套件
  4. 保留原始 PDF 的圖片、文字格式，盡可能讓翻譯後的 PDF 除了語言都接近原始的版本
  5. 製作簡單的 GUI

  這個需求是否足夠詳細，如果不足，請詢問我並補充。
  ```


## 8-2 完整的軟體工程流程

### 第一步：撰寫 PRD（產品需求文檔）

- 輸入的提示內容：

```md
你現在是一位高級 AI 系統架構師。

請協助我撰寫「PDF 翻譯器」的完整 PRD（產品需求文檔）。

要求包含：

1. 以 Python 製作

2. 盡可能支援多種語言翻譯成繁體中文

3. 使用 deep-translator 套件

4. 保留原始 PDF 的圖片、文字格式，盡可能讓翻譯後的 PDF 除了語言都接近原始的版本

5. 製作簡單的 GUI

另外列出需要補充的需求

```

補充PRD細節：
```md

請幫我把這份 PRD 補齊，把所有漏掉的需求補上，按照 PRD格式重新整理分類，補齊邊界情境、錯誤處理、效能、安全性與技術限制。 

不要寫解法，只寫需求。 

```

### 第二步：生成 SDD（系統設計文檔）

- 輸入的提示內容：

  ```md
  請根據這份 PRD，產生一份 SDD（系統設計文件）。

SDD 請包含：
- 系統架構簡介
- 模組拆分
- 每個模組的功能說明
- 資料流程（主要流程說明即可）
- 主要方法與其輸入/輸出
- 錯誤處理重點

請不要寫程式碼，只產生「系統怎麼設計」的內容。

  ```

### 第三步：依照SSD逐步實作
```md
請依照 SDD，開始為這個模組補上完整實作。  
要求：

- 嚴格遵守 SDD 的方法、參數、資料流
    
- 僅在這個檔案內實作功能
    
- 保持程式碼乾淨、可讀
    
- 出問題要顯示例外訊息
    
- 不要變更架構
    

完成一個模組後請停止，等我說下一個。
```


讓 Cursor 自動進行**整合測試**並修正所有問題：
```md
請分析整個專案，檢查所有模組之間的整合問題，包括：

- import 錯誤

- class 或方法名稱不一致

- 參數未對齊

- 路徑錯誤

- 資料流不符合 SDD

請自動修正所有問題，必要時同步更新多個檔案。 

完成後告訴我你修了哪些地方。
```


## 8-3 修正與優化過程

### 修正格式保留問題

- 輸入的提示內容：

```md
PDF 輸出結果有問題，請你：
1. 自動比對 input/output 差異（字數、頁面、區塊）
2. 判斷是哪個模組造成問題（Parser / Translator / Renderer）
3. 提出修復方案
4. 自動修正相關模組

不要重寫專案架構，只修改必要的地方即可。

```
- 使用時機：
  - 修正 bug 時
  - 優化功能時
  - 添加新功能時
  - 任何可能影響整體架構的修改

##  常用提示詞模板

### 新增功能

```md
請在 [模組名稱] 中新增 [功能描述]。

要求：
1. [具體要求 1]
2. [具體要求 2]
3. [具體要求 3]

請不要重寫專案架構，只修改必要的部分。
```

### 修正 Bug

```md
[描述問題現象]

預期行為：[應該如何運作]
實際行為：[目前如何運作]

請修正此問題。

請不要重寫專案架構，只修改必要的部分。
```

### 優化性能

```md
請優化 [模組/功能] 的性能。

當前問題：
- [問題 1]
- [問題 2]

優化目標：
- [目標 1]
- [目標 2]

請不要重寫專案架構，只修改必要的部分。
```




---

提示詞的品質直接決定了 Cursor 生成代碼的品質。花時間寫好提示詞，比花時間除錯更有效率。
